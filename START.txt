  docker-compose up -d                       # 启动生产环境
  docker-compose -f docker-compose.prod-nginx.yml up -d  # 生产+Nginx

管理容器：
  docker ps                                  # 查看容器
  docker logs <容器名>                       # 查看日志
  ./docker-build.sh dev logs -f              # 持续查看日志
  ./docker-build.sh dev shell                # 进入容器

停止容器：
  ./docker-build.sh dev down                 # 停止并移除容器
  docker-compose down -v                     # 停止并删除数据

💡 关键概念（快速理解）

【Docker】
  - 容器化技术，打包应用和依赖
  - 保证"在我的电脑上运行，到任何地方都运行"

【docker-compose】
  - 定义多容器应用的编排工具
  - 简化容器管理，一个文件定义整个应用栈

【版本升级】
  - docker-compose 1.29.2 → docker compose v5.0.2
  - 移除过时的配置字段
  - 完全向后兼容

【多环境配置】
  - 开发环境：快速迭代，代码热加载
  - 生产环境：性能优化，安全加固
  - 可选 Nginx：反向代理，负载均衡
